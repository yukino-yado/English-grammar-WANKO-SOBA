# 英文法わんこそば v3.3

## v3.3の主な変更

- 教師用アプリに「問題を編集する」を追加しました。
- `/question-editor.html` から、既存問題を含む全単元の日本語文・英文を編集できます。
- 編集対象は日本語文と英文のみです。穴埋め、答え、選択肢、解説、単元、難易度は自動生成側の設定を保ちます。
- 編集済み問題は教師側のシステム上で強調され、生徒側には編集済み表示を出しません。
- 編集内容は自動生成時に上書き反映されます。
- ページ間リンクは `/teacher.html`、`/question-editor.html`、`/index.html` の絶対パスでつないでいます。

- 教師用データをVercel Blobに保存し、どの端末・どのブラウザから入っても同じ編集データを共有できるようにしました。
- 教師用・問題編集・生徒用は起動時に共有データを読み込みます。
- 教師用で保存した内容は、同じVercelプロジェクト内の共有データへ保存されます。
- `/api/teacher-package` を追加しました。
- `package.json` を追加し、Vercel Blob用に `@vercel/blob` を使います。

## 画面

- 生徒用：`/index.html`
- 教師用：`/teacher.html`
- 問題編集：`/question-editor.html`

## 反映方法

Vercel Blob を設定した状態でデプロイすると、教師用・問題編集画面で保存した内容は共有データへ保存されます。別端末・別ブラウザでも同じURLへアクセスすれば、共有された編集データを読み込めます。

ローカルで開く場合や Vercel Blob 未設定の場合は、従来通りブラウザ内保存と配布用データで確認できます。


HTML / CSS / JavaScript だけで動くブラウザアプリです。

## 起動方法

ZIPを解凍して、以下のファイルをブラウザで開きます。

- 生徒用：`index.html`
- 教師用：`teacher.html`

## 教師用から生徒用への反映

同じフォルダ内で `teacher.html` と `index.html` を開く構成です。

### 同じブラウザで確認する場合

1. `teacher.html` を開く
2. お品書きや問題を追加する
3. 「生徒版へ反映」を押す
4. `index.html` を開く、または開き直す
5. 生徒用の「先生からの一杯」に追加問題が反映される

### 別端末・別ブラウザで使う場合

1. 教師用で「配布用データを作る」
2. 生徒端末側で teacher.html を開き、配布用データを取り込む
3. 同じ端末で index.html を開く
4. 生徒用に追加問題が反映される

※v3.3では Vercel Blob を使った共有保存に対応しました。Vercel側でBlob Storageを作成し、`BLOB_READ_WRITE_TOKEN` が有効な状態でデプロイしてください。

## Vercel Blob設定

1. Vercelのプロジェクトを開く。
2. StorageからBlobを作成する。
3. 作成時に `BLOB_READ_WRITE_TOKEN` をこのプロジェクトのEnvironment Variableとして有効にする。
4. GitHubへv3.3のファイルを上書きして再デプロイする。
5. 教師用で保存すると、共有データへ保存されます。

共有保存が未設定の場合、教師用画面に「共有データを読み込めませんでした」と表示されます。その場合でも、その端末のブラウザ内保存は残ります。
